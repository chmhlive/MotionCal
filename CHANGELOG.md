# CHANGELOG

本文档记录了自上游官方代码（PaulStoffregen/MotionCal）Fork 以来的主要功能重构与优化。

## 核心架构重构
- **通信协议全面升级为 BLE HID**：将原本基于标准串口（Serial Port）的原始字节流通信，彻底改写为基于 `hidapi` 的低功耗蓝牙（BLE HID）通信。
- **自定义 HID 帧解析**：废弃了原有的串口数据截断与校验机制，新增了一套专门针对 BLE HID Report 协议的解析和发送模块。
- **构建系统重置**：适配了适用于 Linux/WSL 环境的交叉编译脚本，支持通过 MinGW-w64 直接增量构建基于 Windows 的 `MotionCal.exe`。

## 功能特性增强
- **图形化错误提示闭环**：完善了 GUI 异常交互。在面临底层蓝牙设备掉线、连接异常，或写入数据读回校验不匹配时，自动弹出显著的错误警告弹窗，替代了原版仅改变图标状态的微弱提示。
- **拥塞控制与抗丢包机制**：针对蓝牙信道带宽极窄的特性，在批量下发庞大的磁校准矩阵之前，主动向硬件下发暂停采集指令，并引入安全等待期（排空设备及 PC 端缓冲），确保配置指令享有纯净无干扰的传输信道，彻底解决了连续写入时的蓝牙丢包问题。
- **宏定义规范化**：将原版 UI 中用于判定“是否允许下发校准数据”的大量硬编码（Magic Number）全部提取为清晰的宏定义，降低了后续二次开发的维护成本。

## Bug 修复
- 修复了原版日志模块在提取路径时的指针越界与未定义行为（UB）。
- 修复了蓝牙设备连接初始化失败时，释放句柄导致的冗余重连造成的程序假死漏洞。
- 修正了部分源文件的多级缩进与排版风格不统一问题。
